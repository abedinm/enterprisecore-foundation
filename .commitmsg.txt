Add TOTP 2FA, task comments, and Vitest frontend tests

Two-factor authentication (TOTP):
- New TwoFactor model stores secret + enabled flag + JSON-encoded
  bcrypt-hashed backup codes.
- POST /2fa/enroll  -> returns secret + otpauth URI + inline SVG QR.
- POST /2fa/verify  -> consumes the user's first code, enables 2FA,
  returns 8 single-use backup codes (shown once).
- POST /2fa/disable -> requires a current code; backup codes do not
  work here for security reasons.
- GET  /2fa/status  -> simple {enabled: bool} for the UI.
- /auth/login gates token issuance behind a 6-digit TOTP or 8-char
  backup code when 2FA is enabled. Missing code returns 403 with
  "Two-factor code required"; the frontend uses that to switch into
  the second-stage code prompt.
- Backup codes are single-use: a successful login with one removes
  that hash from the user's list.

Task comments:
- New TaskComment model (task_id + author_id + body + edited_at).
- GET    /tasks/{id}/comments
- POST   /tasks/{id}/comments
- PATCH  /tasks/{id}/comments/{cid}  (author only)
- DELETE /tasks/{id}/comments/{cid}  (author OR admin/manager)
- Visibility follows project visibility — same _can_access_task
  rule the rest of /tasks uses.

Frontend:
- New TwoFactor page renders the QR, walks through enroll -> verify
  -> backup-code reveal, supports disable.
- Login page is now two-stage. When the API replies "Two-factor code
  required", the form swaps to a code prompt without leaking that the
  password was correct via a redirect/url change.
- Sidebar gets a "2FA" link visible to every role.

Frontend testing:
- Vitest + jsdom + Testing Library set up.
- 13 tests covering the auth and theme stores plus the UI primitives
  (Button, Badge, Input, Card). Run with `npm test`.
- Wired into CI as a new step on the frontend job.

Verified: 69 pytest + 13 Vitest, all green.
